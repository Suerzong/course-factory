# 阶段 B：结构读取

> 摘自 guide/guide.md 第 701-706 行。本文件是该阶段的完整执行指令。

## 需要的变量

无额外变量。使用阶段 A 收集的变量。

## 前置检查

执行本阶段前，读取 `COURSE_DIR/pipeline-progress.md`：
- 确认阶段 B 状态为"待执行"
- 如果状态为"已完成"，跳过本阶段，直接进入阶段转换
- 如果状态不是"待执行"也不是"已完成"，报错并停止

## 执行指令

Claude 读取 `COURSE_DIR/textbook/chapters/总教材.md`，识别章节结构：
- 按一级标题（`#`）切分
- 输出：章节数量、每章标题、每章起止行号
- 将结构信息展示给用户，**等待用户回复确认**（必须收到用户明确确认后才能继续）

## 质量门

- 识别出的章节数量 ≥ 1
- 每章有明确的标题和起止行号
- 用户确认结构无误

## 完成记录

质量门通过后，更新 `COURSE_DIR/pipeline-progress.md`：
- 将阶段 B 状态从"待执行"改为"已完成"
- 记录完成时间
- 记录识别出的章节数量和每章标题

## 阶段转换

完成记录写入后：
1. 读取 `COURSE_DIR/pipeline-progress.md`
2. 找到第一个状态为"待执行"的阶段
3. 如果找到 → 读取对应的 `guide/pipeline/stage-{x}.md`，执行该阶段
4. 如果没有"待执行"阶段 → 所有阶段完成，进入清理协议
