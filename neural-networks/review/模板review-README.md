# 复盘层说明

`review/` 负责错题、误区和复测记录。它只记录影响 {{COURSE_SHORT_NAME}} 教材理解的问题，不记录代码实现、框架操作、工程调试或部署问题。

## 文件职责

- `mistakes.md`：错题与误区主表。

## 记录原则

- 只记录已经在当前教材路线中讲过的内容。
- 每条误区必须指向 {{COURSE_SHORT_NAME}} 位置和教材段落。
- 错误原因要写成"为什么理解错"，不是只写错题答案。
- 每条误区都应有纠正任务和是否复测通过。
- 正式练习、补救题或再测题中出现核心知识点概念混淆、公式含义错误、概念边界错误，并进入补救时，必须写入 `mistakes.md`。
- 即使补救后当场通过，也必须写入 `mistakes.md`，并将 `是否已复测通过` 标为 `是`，不能只在学习日志中说明。
- 初始诊断中的错误若没有进入正式练习或补救，可以只写入学习日志；一旦同类错误在正式练习中复现，就必须进入 `mistakes.md`。

## 不记录内容

- 旧外部教程造成的工程错误。
- 代码实现、框架 API、部署或工具链错误。
- 当前教材尚未讲到的大块后文内容。
