# {{COURSE_SHORT_NAME}} 任务生成规则

本文件只服务当前课程根目录下的 {{COURSE_SHORT_NAME}} 主课。任务生成必须遵守 `course-rules.md` 的 `source_mode: textbook-only`。

## 源隔离

任务只能来自：

- 当前 `progress/current-position.md` 指向的教学单元。
- `learning-path/chapter-XX.md` 中已解锁的前置单元。
- 对应 `knowledge/teaching-guides/.../*.teaching.md` 中列出的核心知识点和重要背景。
- 对应 `textbook/chapters/*.md` 的教材段落。
- 当前章节的课本习题，且必须确认依赖概念已讲。

不得使用外部教材、旧教程、代码样例、工程项目、框架文档或部署专题。

## 三层标注规则

| 层级 | 任务生成规则 |
|---|---|
| 核心知识点 | 正式题主体。可生成概念解释、公式含义、小推导、课本习题拆解、常见误解纠正。 |
| 重要背景 | 只能轻量问"为什么重要""和核心概念是什么关系"。不得生成复杂推导或章测大题。 |
| 了解即可 | 默认不出正式题。最多课堂中问一句"它属于哪类内容"，不计入掌握度。 |

如果一个"了解即可"内容还没在教材后文正式讲授，禁止提前升级为考点。

教学指引中的 `出题权限` 必须按受控值执行：

| 出题权限 | 处理方式 |
|---|---|
| 正式题 | 可进入立即练习、课后题、章测和掌握度推进依据。 |
| 课堂识别不计分 | 只允许作为讲解中的轻量确认，不写入正确率。 |
| 否 | 不生成题目。 |

## 每道题的强制标注

每道正式题必须带五行边界标注：

```md
- 对应单元：`chXX-...`
- 考察来源：核心知识点 / 重要背景 / 已学前置 / 课本习题
- 教材段落：`¶XXXX` 或 `¶XXXX-¶XXXX`
- 知识点层级：核心知识点 / 重要背景 / 了解即可
- 未学内容检查：无新增大概念 / 已给最小定义 / 推迟到后续章节
```

如果无法填写这五行，说明题目不合格，必须重写。

## 允许题型

| 类型 | 目标 |
|---|---|
| 概念复述题 | 确认能用自己的话说清定义 |
| 概念边界题 | 区分相近概念 |
| 公式含义题 | 理解符号和公式目的 |
| 小推导题 | 跟随教材推导关键一步 |
| 课本习题拆解题 | 把课本习题拆成可完成步骤 |
| 课本例子复述题 | 用教材例子复述概念 |
| 前后关系题 | 串联已学小节 |
| 常见误解纠错题 | 发现并纠正错误理解 |

禁止题型：

- 框架或库使用题。
- 代码实现题。
- 工程调试题。
- 外部数据集任务。
- 部署、压缩、推理链路题。
- 需要查外部资料才能完成的题。
- "了解即可"人名、年份、工具列表重考题。

## 任务前讲解

生成任务前必须先给"本次知识点"：

1. {{COURSE_SHORT_NAME}} 章节号和标题。
2. 本次知识点层级。
3. 一句话定义。
4. 教材为什么在这里讲它。
5. 需要的已学前置。
6. 关键公式或结构；没有公式时写"本节无核心公式"。
7. 公式中每个符号含义；没有公式则解释关键术语。
8. 和相近概念的区别。
9. 本轮只要求掌握到什么边界。

不得加入教材外扩展。若学习者问外部内容，先标记"不属于本轮 {{COURSE_SHORT_NAME}} 主课"。

讲解和出题都不得把教材没有明说的解释改写成硬规则。类比只能来自教材正文、课本习题或最小数学例子；若使用辅助类比，必须声明"只是帮助理解，不作为考点"。当前单元没有要求掌握的后文算法或术语，不得用于扣分或正式题。

## 未学内容拦截

生成题目前逐题检查：

1. 是否依赖后续章节的大概念？
2. 是否依赖本轮没讲过的公式、模型、定理？
3. 是否必须查外部资料才能完成？
4. 是否把"了解即可"层内容当成正式考点？

处理规则：

- 小术语：给一句最小定义，再用轻量理解题。
- 大概念：推迟到对应章节。
- 必要前置：先补正式教学，再出题。
- 外部内容：删除题目。

## 难度控制

| 当前等级 | 可用题型 |
|---:|---|
| 0-1 | 概念复述、边界判断、教材段落复述 |
| 2 | 公式含义、关系解释、轻量课本习题拆解 |
| 3 | 小推导、同源变式、错因分析 |
| 4 | 完整课本习题、跨小节串联 |
| 5 | 讲解纠错、章节结构重构、前后文联系 |

难度只能由当前 {{COURSE_SHORT_NAME}} 掌握证据决定，不能因为学习者过去做过代码或项目而提高。

## 出题后自检

| 检查项 | 要求 |
|---|---|
| 源可追溯 | 每题能指向 {{COURSE_SHORT_NAME}} 小节或教材段落 |
| 层级匹配 | 核心重点考，背景轻量问，了解不正式考 |
| 前置已讲 | 不考尚未教学的大块内容 |
| 术语已定义 | 新术语已给最小定义 |
| 公式可达 | 所需公式已在题中或讲解中出现 |
| 无外部混入 | 不出现外部实操、部署、框架任务 |

若任一项不通过，必须重写任务。

## 错题入库规则

- 正式练习、补救题或再测题中，学习者答错核心知识点或暴露核心概念边界混淆时，必须把该错误写入 `review/mistakes.md`。
- 补救后答对不取消入库；应在错题条目中记录纠正任务和 `是否已复测通过：是`。
- 只有初始诊断中的错误可以不入库，但若同类错误在正式练习中再次出现，必须入库。
