# 学习会话日志

`logs/learning-sessions/` 用于记录每次完整学习闭环。它不替代 `progress/`，只保存过程证据。

## 命名建议

```text
YYYY-MM-DD-chXX-unit-id.md
```

示例：

```text
2026-05-23-ch01-00.md
```

## 使用规则

- 每次学习结束后记录一次。
- 必须使用上方完整模板的标题结构，不得用“读取依据/诊断结果/练习结果”替代模板标题。
- 只记录当前 NNDL 主课证据。
- 不把旧实操经验、代码项目或外部资料写入当前会话结论。

## 验收清单

学习日志合格前必须逐项确认：

- [ ] 包含 `## 本次入口`、`## 本次读取`、`## 教学覆盖`、`## Two Sigma 循环记录`、`## 更新记录`、`## 下次入口`。
- [ ] `Two Sigma 循环记录` 下包含 1-10 步的小标题。
- [ ] 记录正式题范围、题目来源、正确率证据和核心题是否通过。
- [ ] 若有补救，记录补救目标、再测题来源、再测正确率和推进依据。
- [ ] `更新记录` 写明 `current-position.md`、`mastery-tracker.md`、`review/mistakes.md` 的处理结果。
- [ ] 若正式练习、补救题或再测题出现核心知识点错误，`review/mistakes.md` 必须有对应条目；日志不得写“无需新增”作为替代。
- [ ] `下次入口` 能让下一轮直接定位到下一单元。
